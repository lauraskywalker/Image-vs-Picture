IMAGE VS. PICTURE


IMAGE

- img-Element zur Einbindung von Grafikdateien
- "Standalone-Tag" ohne End-Tag
- im XHTML wird der schließende Schrägstrich (/) benötigt


PFLICHTANGABEN

- src (source): für die Referenz der Bilddatei
- alt (alternative): Alternativtext, der angezeigt wird, wenn das Bild nicht geladen werden kann oder soll


OPTIONALE ANGABEN

- height: Angabe der optionalen Höhe (empfehlenswert)
- width: Angabe einer optionalen Breite (empfehlenswert)
- sizes: kommagetrenne Liste mit einem Medienmerkmal und einer Größenangabe
- srcset: eine Kommagetrenne Liste mehrerer URLs mit optinaler Angabe der Breite und Pixeldichte - usemap: zeigt an, dass das Bild Teil einer verweissensitiven Grafik ist
- crossorigin
- usemap


WEITERE INFOS

- Chrome und Safari zeigen den Alternativtext nicht an
- Basic Support für jeden Browser
- srcset wird nicht vom Internet Explorer unterstützt



PICTURE

- Container mit dem mehrere Bildquellen referenziert werden können
- es können mehrere Bildversionen für verschiedene Pixeldichten, Auflösungen und Bildformate zur Verfügung
gestellt werden, aus der der Browser die passende Grafik auswählt
- NUR ein Container für das darin enthaltene img-Element und evtl. vorhandene Source-Elemente


PFLICHTANGABEN

- keine oder mehrere Source-Elemente
- img-Element (inkl. src und alt)


OPTIONALE ANGABEN

- media: erlaubt die Angabe eines Media-Query, das der User-Agent auswertet, um das gewünschte Source-Element auszuwählen
- type: erlaubt die Angabe eines MIME-Typs für Ressourcen, die im Attribut srcset des Source-Elements angegeben wird


WEITERE INFOS

- kein Basic Support für Safari
